Understanding HDMI & Browser Limitations

Important: Browser Capabilities

Web browsers (and therefore this JavaScript-based tool) operate in a sandboxed environment for security and privacy. They **CANNOT** directly access or query low-level hardware information such as:

  • The specific HDMI version (e.g., 1.4, 2.0, 2.1) of your graphics card, display, or cable.
  • The bandwidth capacity or "speed" rating of your HDMI cable.
  • The actively negotiated HDCP version (e.g., 1.4, 2.2, 2.3) on the physical link.

This information is handled at the Operating System, driver, and physical hardware levels, well below what is exposed to web pages.

Maximum Display Resolution (Reported)
-

The screen resolution reported by the browser (e.g., `screen.width` x `screen.height`). While high resolutions (like 4K) often require newer HDMI standards (e.g., HDMI 2.0+ for 4K@60Hz), this value reflects what the OS/driver tells the browser about the *display*, not necessarily the full capability of the cable or HDMI port itself.

HDCP Status (EME API Hint)
-

This tests if the browser's Encrypted Media Extensions (EME) can satisfy a policy requiring HDCP for a common DRM (e.g., Widevine). A "supported" status suggests the browser *believes* HDCP is available for protected content playback. It does not confirm the specific HDCP version active on the wire.

What Can This Tool Tell You (Indirectly)?

While direct HDMI hardware details are out of reach, this tool can show related information:

  • Screen Resolution & Color Depth: Provides clues about the data rate being currently utilized.
  • HDR Support (CSS Media Query): Indicates if the browser *thinks* the display can render HDR content.
  • EME/DRM Check for HDCP: Suggests if the software chain is prepared for HDCP-protected playback.
For definitive HDMI port/cable/version details, please consult your device specifications or use OS-level diagnostic tools.
